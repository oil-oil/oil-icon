# 图标生成 · 设计与工程规范

生成风格一致、可稳定切割的图标集的通用做法与规则。只写通用原则，不含具体项目案例。

## 一、工具约束

- 常见的 gpt-image 路径（Codex / OpenAI Images）只接受 `prompt` 文本、不能喂参考图 → 风格一致靠统一提示词；画图工具按能力自适应选择（见 `reference/image-providers.md`），稳定切割靠切图脚本。
- 输出是位图 PNG（约 1254×1254 方图），不是矢量；尺寸 / 透明 / 张数不可直接指定，全在 prompt 里描述。
- 位图 hold 不住"系统级"图标（见第五节）。

## 二、构图与切图

- 一张 4×4 sheet 一次生成 16 个：同一次生成里风格天然统一，也省成本。图标多用 4×4；追求切割稳定用 3×3（9 个，格子更大、间距更足、更好切）。
- 生成背景固定纯灰 `#808080`；提示词强制无文字、无网格线、图标间留足间距、不投影到背景。
- 切图脚本 `slice_icons.py` 流程：
  1. 按网格分格，每格略微外扩（~6%）兜住偏移的图标。
  2. 每格抠背景。
  3. 连通域清理：只保留居中的主图标 + 紧挨它（间隙 <5%）的部件，隔着间距的邻格碎片一律丢弃——避免把相邻格图标的一部分带进当前格。
  4. bbox 裁剪 + 居中到方形画布 + resize。
- 抠背景基准用固定 `#808080`，不逐格采样四角：大图标或带底色瓷砖会让角落落在图标 / 瓷砖上，导致背景被误判、把底色误删。
- flood-fill 删除所有接近灰底的像素，含图标内部镂空（放大镜镜片 / 齿轮中心 / 日历格子 / 锁孔），否则这些洞残留灰、图标发脏。
- 两条抠图管线，由每个 style 的 `cutout` 字段指定：
  - 硬边（线性 / 面性 / 撞色 / 等距 / 扁平 / 带底色瓷砖）→ 固定灰底 flood-fill（thresh ~26–34）。
  - 软边 / 低对比 / 反光 / 照片（卡通 / 3D / 贴纸 / 照片实物 / 白色物体 / 徽章）→ rembg（u2net ML 抠图），软边和投影都能干净去除。

## 三、一致性

- 机制：统一提示词。
- 调色板必须写死 hex，否则彩色风格会漂色；低饱和品牌色尤其会被拉艳。
- 可选：锚点法——新 sheet 里掺入已定稿图标，把风格拽回来。

## 四、风格规则

- 视觉重量层级（通用，最重要）：图标的"定义元素"（线性风格的线、面性风格的实心轮廓）必须始终是视觉最重的；点缀 / 第二色 / 细节的重量要压在它之下。视觉重量 ≈ 与底色的明度反差 × 面积 × 饱和度——线越细，点缀就得越浅越小。给细线配又深又大的实填会重量倒挂：线沦为框、深块抢焦点，整套就不像那个风格了。点缀优先用比线更浅的淡色 / 半透明、小到中面积；可用柔和渐变或材质增加"库级"质感，渐变也让点缀更轻。
- Duotone 双色：两色必须有反差（一深一浅拉开明度，或色相明显不同）；相近的低饱和两色会糊成一团、层次全丢。第二色语义化只点一个关键部件，不是每块形状随便刷两色。
- Solid 面性：实心是大色块，铺满一屏显脏。内容密集 / 编辑调性 UI 默认用 line 或 line + 极小点缀；solid 只用于小而稀疏的位置（tab bar、单个强调）。
- 面性单色需在 prompt 写死 hex，避免自作主张上多色。
- 卡通 / 3D 的柔和投影：prompt 写 no drop shadow 只减轻不根治，走 rembg。

## 五、"设计感"

核心：给通用图标染品牌色 ≠ 品牌图标系统。缺设计感的根因：

1. 隐喻太通用直白（人+闪光=创作者、画框=作品、领奖台=排行榜）——没有品牌自己的视角 / 母题，换个色就是别家的。
2. 没有统一构造系统——图标系统是一套构造规则（同网格、同线宽、同圆角、同收笔、同"繁简预算"），一套图标共享同一副骨架。位图逐格徒手画会飘，读作"一堆相似但不成体系"。
3. 繁简不一致——抽象程度要统一。
4. 太写实不做减法——好图标把一个意思减到只剩本质。
5. 缺品牌母题 signature——一个反复出现的记号（呼应品牌圆角 / 字体 / 容器形），让人一眼认出是谁的。

补设计感：spec 先定构造系统（网格 + 线宽 + 圆角 + 收笔 + 繁简预算），而非只写"线性 + 某色"；定一个品牌母题让它复现；做减法；第二色语义化。真要成系统：位图出方向，选定后矢量重建 / 归一化做终稿——位图负责探索，不负责终稿的严谨。

## 六、架构

- 两层：内置风格库（每个 = preamble + 锁死 palette + cutout 管线 + thresh + 构造系统）；用户适配层（从项目设计 token 反推 spec / 基于内置风格改 / 纯文字描述）。
- spot vs UI 边界：位图生成图标用于 spot / 卡片头 / 营销位 / 空状态 / 有性格的地方；密集功能 UI（16–24px）用矢量图标库。
- 软边 / 低对比风格走 rembg。
